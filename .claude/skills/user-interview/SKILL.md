## 목적

유저 인터뷰 설계부터 실행, 인사이트 도출까지 전 과정을 구조적으로 지원한다.

## 입력 변수

- `연구 목적`: 무엇을 알고 싶은가? (가설 또는 질문)
- `타겟 사용자`: 인터뷰할 사용자 프로필
- `제품 단계`: 발굴 / 검증 / 개선
- `인터뷰 수`: 계획 중인 인터뷰 횟수

## 프레임워크

### 1. 연구 목적 및 핵심 질문 설정
- 이 인터뷰로 답해야 하는 핵심 질문 2~3개를 먼저 정한다
- 가설: "우리는 [사용자]가 [문제]를 경험한다고 가정한다"
- 검증 기준: 몇 명이 같은 패턴을 보이면 가설을 수용할 것인가?

### 2. 스크리너 (참가자 선별 기준)
| 항목 | 포함 기준 | 제외 기준 |
|------|-----------|-----------|
| 직업 | ... | 경쟁사 직원 |
| 경험 | [관련 경험 有] | [관련 경험 無] |
| 행동 | [목표 행동 수행] | ... |

### 3. 인터뷰 구조 (60분 기준)
| 단계 | 시간 | 질문 유형 |
|------|------|-----------|
| Warm-up | 5분 | 현재 역할, 일상 파악 |
| 현재 상황 탐색 | 15분 | 어떤 상황에서 이 문제를 만나는가? |
| 문제 깊이 파기 | 20분 | 가장 불편했던 순간? 어떻게 해결했나? |
| 이상적 경험 | 10분 | 어떻게 바뀌면 가장 좋겠나? |
| 마무리 | 10분 | 추가 의견, 추천인 요청 |

### 4. 핵심 질문 원칙
- 개방형 질문 사용 ("어떻게", "왜", "말씀해주세요")
- 유도 질문 금지 ("~하시죠?" 형태 피하기)
- 특정 솔루션에 대한 의견 묻지 않기 (검증 단계 전)
- 행동 기억 질문 선호: "마지막으로 [행동]을 했을 때를 말씀해주세요"

### 5. 인사이트 도출 (어피니티 다이어그램)
1. 인터뷰 내용에서 핵심 발언 추출 (Post-it 형식)
2. 유사한 발언끼리 그룹핑 (상향식)
3. 각 그룹에 라벨 붙이기 (2~3단어)
4. 패턴에서 인사이트 도출: "사용자들은 [상황]에서 [행동]한다. 왜냐하면 [이유] 때문이다."

## 출력 형식

인터뷰 가이드를 다음 섹션으로 제공한다:
1. 인터뷰 목적 및 핵심 질문
2. 스크리너 기준표
3. 본 인터뷰 질문 목록 (단계별)
4. 관찰 체크리스트
5. 인사이트 기록 양식

## 품질 체크리스트

- [ ] 인터뷰 목적이 명확한 가설로 정의되었는가?
- [ ] 질문이 개방형이고 유도 질문을 포함하지 않는가?
- [ ] 행동 기억 질문을 포함하는가?
- [ ] 인터뷰 시간이 참가자 피로도를 고려했는가?
- [ ] 인사이트 기록 및 분석 방법이 정의되었는가?
