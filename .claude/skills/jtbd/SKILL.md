## 목적

고객이 제품을 '고용'하는 진짜 이유(Job)를 발굴하고, 구체적인 Job Statement로 표현하며, 전환 동기를 이해한다.

## 입력 변수

- `제품/서비스명`: 분석 대상
- `대상 고객`: 인터뷰하거나 분석할 사용자
- `관찰된 행동`: 고객이 하고 있는 것 (선택)
- `경쟁 제품`: 고객이 현재 사용 중인 대안

## 프레임워크

### 1. Job의 3가지 유형
| 유형 | 정의 | 예시 |
|------|------|------|
| 기능적 Job | 실질적으로 해결해야 할 과제 | "회의 일정을 잡는다" |
| 감정적 Job | 느끼고 싶거나 피하고 싶은 감정 | "효율적인 사람으로 보이고 싶다" |
| 사회적 Job | 타인에게 어떻게 인식되고 싶은가 | "팀에서 신뢰받는 PM이 되고 싶다" |

### 2. Forces Diagram (전환 결정 4가지 힘)
```
현재 상황의 Push (밀어내는 힘)
→ "현재 솔루션의 불만족: ..."

새 제품의 Pull (끌어당기는 힘)
→ "새로운 제품의 매력: ..."

불안(Anxiety, 새 제품 채택을 막는 힘)
→ "전환 비용, 학습 곡선, 불확실성: ..."

관성(Habit, 현상 유지를 선호하는 힘)
→ "현재 방식에 익숙함, 변화 저항: ..."
```

### 3. JTBD 인터뷰 구조
**목표**: 고객이 제품을 처음 구매/사용 결정한 순간을 재구성한다.

핵심 질문 흐름:
1. "처음으로 [제품]을 사용하기로 결정한 때를 기억하시나요?"
2. "그 결정을 하기 전에 어떤 상황이었나요?" (상황)
3. "왜 그때 [행동]을 하셨나요?" (동기)
4. "결정을 망설이게 한 것이 있었나요?" (불안)
5. "그전에는 어떻게 이 문제를 해결하셨나요?" (현재 솔루션)
6. "이제 [제품]을 통해 어떤 변화가 생겼나요?" (결과)

### 4. Job Statement 작성 형식
"나는 [맥락/상황]에서, [목표/동기]를 이루기 위해, [행동]을 원한다."

또는 더 확장된 형식:
"When [상황], I want to [동기/목표], so I can [기대 결과]."

예시: "분기 사업 계획을 세울 때, 팀의 우선순위를 명확히 정렬하고 싶다. 그래야 낭비 없이 핵심에 집중할 수 있기 때문이다."

### 5. 기회 점수 계산
기회 점수 = 중요도 평균 + max(중요도 평균 - 현재 만족도 평균, 0)

점수가 높은 Job = 현재 솔루션이 잘 해결하지 못하는 중요한 Job → 제품 기회

## 출력 형식

1. Job Statement (기능/감정/사회 각 1개)
2. Forces Diagram 분석
3. JTBD 인터뷰 질문 목록
4. 기회 점수 테이블 (Job × 중요도 × 만족도)

## 품질 체크리스트

- [ ] Job이 솔루션이 아닌 고객 목표로 표현되었는가?
- [ ] 기능/감정/사회 3가지 유형을 모두 탐색했는가?
- [ ] Forces Diagram에 4개 힘이 모두 포함되었는가?
- [ ] Job Statement가 "When / I want / So I can" 구조를 따르는가?
- [ ] 인터뷰 질문이 유도 질문 없이 개방형으로 작성되었는가?
