## 목적

조직·팀·개인 수준의 OKR을 올바르게 설계하고 달성률을 분석하는 전 과정을 지원한다.

## 입력 변수

- `조직 수준`: 회사 / 팀 / 개인
- `기간`: 분기 / 반기 / 연간
- `주요 전략 방향`: 이번 기간 핵심 우선순위
- `현재 OKR (있는 경우)`: 검토 대상 OKR

## 프레임워크

### 1. Objective 작성 원칙
좋은 Objective의 특징:
- 영감을 주고 방향을 명확히 한다
- 측정 지표를 포함하지 않는다 (측정은 KR의 역할)
- 기간 내 달성 가능하다
- 조직의 전략적 우선순위를 반영한다

나쁜 예: "매출을 올린다" → 좋은 예: "국내 1위 AI PM 도구로 자리잡는다"

### 2. Key Result 작성 원칙
좋은 KR의 특징:
- 산출 지표가 아닌 결과 지표로 측정 (Output vs Outcome)
- 0~1.0 또는 0~100%로 달성률 표현 가능
- Objective와 인과관계가 명확하다
- 2~5개가 적정 (너무 많으면 집중력 분산)

| 구분 | 나쁜 예 | 좋은 예 |
|------|---------|---------|
| 산출 지표 | 블로그 포스트 10개 발행 | 오가닉 트래픽 월 50,000 달성 |
| 활동 지표 | 고객 미팅 20회 | 고객 유지율 85% 달성 |

### 3. OKR 계층 구조
```
회사 OKR
  └── 팀 OKR (회사 OKR의 KR을 달성하기 위한 방법)
        └── 개인 OKR (팀 OKR 기여)
```
하위 OKR은 상위 OKR에 기여해야 하지만, 완전히 동일할 필요는 없다.

### 4. 달성률 분석 기준
| 달성률 | 해석 |
|--------|------|
| 0.7~1.0 | 목표 달성 (0.7이 기준점) |
| 0.5~0.7 | 부분 달성, 원인 분석 필요 |
| 0~0.5   | 미달성, 심층 회고 필요 |
| 1.0 초과 | 목표 설정이 너무 낮았음 |

### 5. OKR 회고 질문
- 달성한 KR: 어떤 활동이 기여했는가?
- 미달성 KR: 통제 가능/불가능 원인은?
- 다음 분기에 계속할 것 / 중단할 것 / 새로 시작할 것?

## 출력 형식

OKR을 다음 테이블로 제공한다:

| # | Objective | Key Result | 목표값 | 현재값 | 달성률 |
|---|-----------|------------|--------|--------|--------|
| 1 | ... | KR1: ... | ... | ... | ... |
| 1 | ... | KR2: ... | ... | ... | ... |

## 품질 체크리스트

- [ ] Objective가 측정 지표를 포함하지 않는가?
- [ ] KR이 Outcome 지표인가 (Output이 아닌)?
- [ ] 팀당 Objective 2~3개, KR 팀당 2~5개 범위인가?
- [ ] 상위 OKR과 연결 관계가 명확한가?
- [ ] 달성률 0.7이 도전적이지만 현실적인가?
