## 목적

RICE, MoSCoW, WSJF, 기회 점수 등 복수의 프레임워크를 활용해 기능 우선순위를 객관적으로 산정한다.

## 입력 변수

- `기능 목록`: 우선순위를 정할 기능/아이디어 목록
- `팀 규모/역량`: 개발 리소스 맥락
- `전략 목표`: 이번 분기 핵심 목표
- `제약 조건`: 기술 부채, 의존성, 마감일 등

## 프레임워크

### 1. RICE 스코어링
RICE = (Reach × Impact × Confidence) ÷ Effort

| 요소 | 정의 | 단위 |
|------|------|------|
| Reach (도달) | 기간 내 영향받는 사용자 수 | 명/월 |
| Impact (임팩트) | 사용자 1명에게 미치는 영향 | 0.25/0.5/1/2/3 |
| Confidence (확신도) | 추정치에 대한 신뢰도 | 50%/80%/100% |
| Effort (노력) | 전체 팀 개발 기간 | 인월(person-month) |

| 기능 | Reach | Impact | Confidence | Effort | RICE |
|------|-------|--------|------------|--------|------|
| 기능 A | | | | | |
| 기능 B | | | | | |

### 2. MoSCoW 분류
- **Must Have**: 없으면 제품이 동작하지 않거나 출시 불가
- **Should Have**: 중요하지만 없어도 출시 가능 (단기 로드맵)
- **Could Have**: 있으면 좋지만 임팩트 낮음 (장기 검토)
- **Won't Have**: 이번 범위에서 명시적으로 제외

분류 원칙: Must Have는 전체의 60% 이하로 유지한다. 전부 Must이면 우선순위가 없는 것과 같다.

### 3. WSJF (Weighted Shortest Job First)
WSJF = (User Value + Time Criticality + Risk Reduction) ÷ Job Size

SAFe(Scaled Agile Framework)의 우선순위 산정 방식으로, 비용 지연(Cost of Delay) 개념 적용.

| 기능 | User Value | Time Criticality | Risk Reduction | Job Size | WSJF |
|------|------------|-----------------|----------------|----------|------|
| | 1~10 | 1~10 | 1~10 | 1~10 | |

### 4. 기회 점수 (Opportunity Scoring)
Tony Ulwick의 ODI(Outcome-Driven Innovation) 방식.

기회 점수 = 중요도 평균 + max(중요도 - 현재 만족도, 0)
- 중요도가 높고 현재 만족도가 낮은 항목 = 최우선 기회

| Job/기능 | 중요도(1~10) | 현재 만족도(1~10) | 기회 점수 |
|---------|-------------|-----------------|-----------|
| | | | |

### 5. 최종 우선순위 매트릭스
임팩트(높음/낮음) × 노력(낮음/높음) 2×2 매트릭스:
- 임팩트 높음, 노력 낮음: **즉시 실행** (Quick Win)
- 임팩트 높음, 노력 높음: **전략적 투자** (Big Bet)
- 임팩트 낮음, 노력 낮음: **시간 있을 때** (Fill-in)
- 임팩트 낮음, 노력 높음: **제외** (Thankless)

## 출력 형식

1. 적용한 프레임워크 설명 및 선택 이유
2. 각 기능의 점수 테이블
3. 최종 우선순위 순서 (1위~N위)
4. 우선순위 결정 근거 요약

## 품질 체크리스트

- [ ] 우선순위 기준이 전략 목표와 연결되었는가?
- [ ] RICE/WSJF의 가정치가 명시되었는가?
- [ ] Must Have가 전체의 60% 이하인가?
- [ ] 모든 이해관계자가 동의할 수 있는 기준인가?
- [ ] 다음 스프린트에 실행 가능한 형태로 결론 났는가?
