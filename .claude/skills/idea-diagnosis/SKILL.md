## 목적

아이디어를 바람직성·실현 가능성·사업성 3개 축으로 빠르게 진단하고, 검증이 필요한 핵심 가정을 식별한다.

## 입력 변수

- `아이디어 설명`: 제품/서비스의 핵심 개념
- `타겟 고객`: 주요 사용자 또는 구매자
- `해결하는 문제`: 고객의 어떤 불편을 해소하는가?
- `제안 가치`: 기존 대안 대비 나은 점

## 프레임워크

### 1. 3축 진단 (각 1~5점)

**바람직성 (Desirability)**: 고객이 정말 원하는가?
- 5점: 인터뷰/데이터로 검증된 강한 페인포인트
- 3점: 존재하지만 빈도·강도가 낮거나 불명확
- 1점: 가정에만 근거, 검증 없음

**실현 가능성 (Feasibility)**: 기술적으로 만들 수 있는가?
- 5점: 기존 기술로 즉시 구현 가능
- 3점: 일부 기술적 도전 있음
- 1점: 현재 기술로 불가능하거나 매우 어려움

**사업성 (Viability)**: 수익을 낼 수 있는가?
- 5점: 명확한 수익 모델, 단위 경제성 양호
- 3점: 수익 모델은 있지만 검증 필요
- 1점: 수익화 방법 불명확

### 2. 핵심 가정 맵핑
| 구분 | 가정 내용 | 검증 방법 | 우선순위 |
|------|-----------|-----------|----------|
| 고객 존재 | [고객이 이 문제를 가진다] | 인터뷰 5명 | 매우 높음 |
| 문제 심각도 | [현재 해결책에 불만족한다] | 설문 조사 | 높음 |
| 솔루션 수용 | [우리 방법으로 해결된다] | 프로토타입 테스트 | 중간 |
| 지불 의향 | [돈을 낼 의향이 있다] | 가격 제안 테스트 | 높음 |

### 3. 경쟁 대안 분석
| 대안 | 장점 | 단점 | 우리 차별점 |
|------|------|------|------------|
| 대안 1 | ... | ... | ... |
| 대안 2 | ... | ... | ... |
| 아무것도 안 함 | 비용 없음 | 문제 지속 | ... |

### 4. 린 캔버스 요약 (9개 블록)
1. 문제 (Top 3 페인포인트)
2. 고객 세그먼트
3. 고유 가치 제안 (UVP)
4. 솔루션 (핵심 기능 3가지)
5. 채널
6. 수익 구조
7. 비용 구조
8. 핵심 지표
9. 불공정 우위

## 출력 형식

1. 3축 진단 점수판 (레이더 차트 형태로 텍스트 표현)
2. 핵심 가정 테이블 (우선순위 순)
3. 가장 큰 리스크 3가지
4. 즉시 검증 실험 제안 (2주 내 실행 가능)

## 품질 체크리스트

- [ ] 3개 축 점수에 근거가 명시되었는가?
- [ ] 가정이 사실과 명확히 구분되었는가?
- [ ] 검증 방법이 구체적이고 실행 가능한가?
- [ ] "아무것도 안 함"을 포함한 모든 대안을 검토했는가?
- [ ] UVP가 "왜 이것이고, 왜 지금인가?"에 답하는가?
